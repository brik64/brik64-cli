Generate, audit, and publish `BRIK64 CLI Beta17` only after a fresh fixpoint evidence campaign proves the exact claim being made. ## Implementation Rules

1. Do not mutate generated artifacts as source.
2. Any functional change must originate in PCD/polymer and be regenerated.
3. If a manual patch is used during investigation, label it `prototype_non_claim` and re-express it in canonical PCD before release.
4. If byte identity fails, Beta17 remains a candidate and no public fixpoint claim is allowed.
5. If any public channel fails during publish, record `FAILED_PARTIAL_PUBLICATION_INCIDENT` and do not declare the release closed.

## Public Claim Policy

Allowed before evidence closure:

- `Beta17 fixpoint candidate`
- `reproducibility campaign in progress`
- `internal evidence under review`

Allowed only after evidence closure:

- exact, bounded statement supported by the evidence pack, for example: `Beta17 candidate regenerated artifact X byte-identically under environment Y with input hashes Z`.

Not allowed without LEVEL 5 or equivalent bounded evidence:

- `universally correct`
- `formal N5 status`
- `independent of all toolchains`
- `definitive fixpoint` without hash-identical reproduction evidence

## Test Plan

Minimum publication gate: NIVEL 3 for release functionality and NIVEL 4-style reproducibility stress for the bounded fixpoint claim.

- Direct CLI tests: init, doctor, engine status, certify, verify, emit TS/Python/Rust, generated tests, polymerize, lift, monomers, lock, update, telemetry off.
- Adversarial tests: corrupt PCD, missing header, invalid monomer, type mismatch, stale cert, path traversal, symlink traversal, output outside workspace, unsupported lift construct.
- Fixpoint tests: first-pass artifact hash, second-pass artifact hash, environment manifest, serial match, input PCD hash lock, harness report, repeat run under clean workspace.
- Surface tests: curl installer, beta manifest, release manifest, GitHub release, SDK marketplaces, docs, web, skills, llms files, changelog, claim-safe public scan.
