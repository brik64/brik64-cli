Status: staged. npm authentication is validated through a temporary npm
userconfig generated by the guard from the 1Password-backed token.

This runbook is for publishing the beta9 SDK packages only as part of the
atomic beta9 release train. Do not publish PyPI or crates.io independently while
npm is blocked.

## Non-Mutation Preflight

Run this first. It must not publish anything.

```bash
BRIK64_USE_1PASSWORD=1 npm run gate:beta9:sdk-marketplace-publish
```

Expected state when token/userconfig auth fails:

```text
decision=BLOCKED_BETA9_SDK_MARKETPLACE_PUBLISH
publicationMutated=false
failures=npm_auth_failed_or_missing
```

Expected state after npm token and temporary userconfig validation:

```text
decision=PASS_BETA9_SDK_MARKETPLACE_PUBLISH_DRY_RUN
publicationMutated=false
```

## npm Token Requirements

The npm automation/publish token must have publish rights for the `@brik64`
scope.

Store it in 1Password:

```text
vault: BRIK64
item: npmjs.com BRIK64
field: API KEY
```

The guard must read this token through 1Password and create a temporary
`NPM_CONFIG_USERCONFIG` containing only:

```text
registry=https://registry.npmjs.org/
//registry.npmjs.org/:_authToken=${NODE_AUTH_TOKEN}
```

Do not paste the token into the shell, logs, GitHub comments, commits, issue
bodies, or a persistent `.npmrc`. Use the 1Password-backed preflight to verify
it.

## Atomic SDK Marketplace Publication

Only run this after the non-mutation preflight passes.

```bash
BRIK64_USE_1PASSWORD=1 \
BRIK64_RELEASE_CONFIRM="PUBLISH SDK 0.1.0-beta.9" \
npm run publish:beta9:sdk-marketplaces
```

The command order inside the guard is:

1. Build/check JS, Python and Rust packages.
2. Validate npm authentication before any mutation.
3. Confirm PyPI token material is present and package artifacts pass
   `twine check`.
4. Validate crates.io ownership/auth.
5. Publish PyPI, npm, then crates.io, skipping a channel only if the exact
   beta9 version already exists.

If any channel fails, stop. Do not manually publish the remaining channels.
Keep the evidence report and repair the failing channel before continuing.

## Post-Publish Verification

After SDK publication, run:

```bash
npm run gate:beta9:sdk-marketplaces
npm run gate:beta9:manifest-drift
npm run gate:beta9:release-readiness
```

Required before the final public train:

```text
PASS_BETA9_SDK_MARKETPLACES
PASS_BETA9_MANIFEST_STAGED_BLOCKED_ALIGNMENT
BLOCKED_BRIK64_CLI_BETA9_RELEASE_READINESS
```

`PASS_BETA9_MANIFEST_STAGED_BLOCKED_ALIGNMENT` is expected until the active
release manifest is promoted from beta8 public to beta9 public during the final
atomic release train.

## Final Release Train Boundary

The SDK marketplace guard does not publish GitHub release assets, curl/GCP
installer assets, docs, web, changelog, skills, or live verification.

Those surfaces remain controlled by:

```bash
npm run release:train:publish-plan
npm run release:train:publish-execute
```

When the active manifest is promoted to beta9, the publish plan includes
`sdk_marketplace_publish_guard` before public-surface mutations. This prevents
a bad npm token from causing a partial release.

## Fail-Closed Rules

- Do not edit `release/manifest.json` to beta9 public until all beta9 public
  surfaces are published and live-verifiable.
- Do not publish PyPI or crates.io alone while npm is blocked.
- Do not use local `.npmrc`, shell history, or unstaged secrets as evidence.
- Do not claim beta9 is public until `gate:beta9:release-readiness` and live
  verification pass with beta9 public surfaces.
