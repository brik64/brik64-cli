## 2. Confirm Branch And Worktree State

Run from the CLI repo:

```bash
git status --short --branch
git rev-parse HEAD
gh pr view <number> --json url,state,mergeStateStatus,reviewDecision,statusCheckRollup
```

Required result:

- [ ] Worktree has no uncommitted source changes.
- [ ] Generated evidence is either committed intentionally, archived outside the
  release branch, or regenerated by the workflow.
- [ ] PR checks are green.
- [ ] Required review is present.
- [ ] Last-push approval requirement is satisfied.

## 6. Dispatch Publication

Only after the previous sections are complete:

```bash
gh workflow run release-train-publish.yml \
  --repo brik64/brik64-cli \
  --ref main \
  -f manifest_digest="<manifest-sha256>" \
  -f confirm="PUBLISH <version> <manifest-sha256>" \
  -f execute_publication=true
```

## 7. Close With Live Verification

```bash
gh workflow run release-train-live-verify.yml \
  --repo brik64/brik64-cli \
  --ref main
```

Close the release only when the verifier reports:

```text
decision=PASS_RELEASE_TRAIN_LIVE_VERIFY
failures=[]
```
